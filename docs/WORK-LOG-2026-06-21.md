## 1. Poster / image generation

### Layer opacity
- **Issue:** Reducing one layer's opacity darkened the whole poster.
- **Cause:** Text/shape opacity merge filled a full-canvas temp with transparent
  black, then `imagecopymerge()` (which ignores alpha) blended that black over
  every layer.
- **Fix:** Copy the current canvas into the temp first, so opacity stays scoped
  to the single element.
- Files: `app/Services/Poster/TemplateRenderService.php`

### Player photo auto-enhance
- **Request:** Player photos looked low-contrast/washed-out.
- **Done:** Player/captain cut-outs now get a mild, alpha-safe contrast +
  brightness boost during rendering (logos/sponsors untouched). Cached, with
  safe fallback to the original.
- Files: `app/Services/Poster/TemplateRenderService.php`

### Match summary — bowling order
- **Issue:** Bowlers were effectively sorted by economy only (a 2-wicket bowler
  sank below 0-wicket bowlers).
- **Fix:** Sort by **wickets desc**, with **economy** as the tiebreaker.
- Files: `app/Services/Poster/MatchSummaryPosterService.php`

### Match summary — overs showing `0`
- **Issue:** Partial overs like `0.4` displayed as `0`.
- **Cause:** CricHeroes import cast overs to int, dropping the balls.
- **Fix:** Keep the raw cricket-notation overs from CricHeroes.
- Files: `app/Services/CricHeroes/CricHeroesScraper.php`
- **Action still needed:** re-import/re-sync any match scored before this fix to
  correct its stored `overs`. New imports are correct automatically.

## 2. Teams & players

### Add Player error
- **Issue:** Adding a player failed with a generic "An error occurred".
- **Cause:** 2MB image limit + the request didn't ask for JSON, so validation
  failures came back as an HTML redirect.
- **Fix:** Raised image limit to 6MB; request now expects JSON and shows the
  real validation message.
- Files: `app/Http/Controllers/Backend/ActualTeamController.php`,
  `resources/views/backend/pages/actual_teams/edit.blade.php`
- **Server:** PHP `upload_max_filesize` raised to 8MB, `post_max_size` to 12MB.

### Tournaments dropdown filtered by organization
- **Issue:** Team form showed all tournaments regardless of organization; the
  searchable dropdown was awkward.
- **Fix:** Dropdown now shows only the selected organization's tournaments and
  re-filters instantly when the organization changes; non-superadmins are scoped
  to their own org.
- Files: `app/Http/Controllers/Backend/ActualTeamController.php`,
  `resources/views/components/inputs/combobox.blade.php`,
  `resources/views/backend/pages/actual_teams/{create,edit}.blade.php`

### Player photo upload simplified
- Reduced to a single recommended portrait ratio (3:4) for player and captain.
- Files: `resources/views/components/player-image-upload.blade.php`

## 5. Font Manager (new feature)

- **What:** Superadmin-only Font Manager at **Settings → Fonts** to install any
  Google Font or upload custom `.ttf`/`.otf` fonts. Installed fonts appear in the
  template editor dropdown.
- **Key guarantee:** the editor preview and the generated poster use the **same
  TTF file**, so there's no longer a difference between design and output.
- **Existing templates unaffected** — built-in fonts remain; new fonts are opt-in.
- **Notable fix:** Google Font install initially failed because the chosen
  User-Agent made Google serve `.woff` (unusable by GD); switched to a UA that
  returns `.ttf`.
- Files:
  - `database/migrations/2026_06_20_000000_create_fonts_table.php`
  - `app/Models/Font.php`
  - `app/Services/Fonts/FontService.php`
  - `app/Http/Controllers/Backend/FontController.php`
  - `resources/views/backend/pages/fonts/index.blade.php`
  - `app/Services/MenuService/AdminMenuService.php` (Settings → Fonts menu item)
  - `app/Services/Poster/TemplateRenderService.php` (`getFontFile` prefers installed fonts)
  - `resources/views/backend/pages/tournaments/templates/editor.blade.php` (dropdown, @font-face, preload)
- **Tests:** 11 passing — `tests/Unit/FontServiceTest.php`,
  `tests/Feature/FontManagerTest.php`.
- **Server:** `public/fonts/google` and `public/fonts/custom` created and made
  writable by the web user; migration run on production.
